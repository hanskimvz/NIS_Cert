## 1. 일반사항

### 1.1 운용 환경 정의
- 제품의 인가된 관리자는 정기적으로 펌웨어 및 소프트웨어 패치를 수행해야 한다.
- 관리자는 HTTPS, TLS, SSH 등 암호화 통신을 통해 웹브라우저로 접속한다.
- 로컬 콘솔 접속은 불가하며 기본 접속 수단은 웹브라우저이다.

### 1.2 제품 개요
IP카메라는 영상(및 선택적 오디오)을 촬영·압축하여 IP 네트워크로 전송하는
하드웨어 일체형 장비이다.

---

## 2. 영상 보안

### 2.1 영상 프로토콜 인증
- ONVIF, RTSP 등 표준 프로토콜에서 사용자 인증 제공
- Digest 인증 사용 시 RFC 7616 준수
- 암호 알고리즘 보안강도 112bit 이상

### 2.2 영상 전송 보안
- HTTPS, TLS 1.2 이상, SSH v2 사용
- 자체 프로토콜 사용 불가

### 2.3 영상 저장 보안 (조건부 필수)
- 내부 저장 시 KCMVP 암호모듈로 암호화
- 외부 파일 제공 시 암호화 필수

## 3. 식별 및 인증

### 3.1 사용자 식별 및 인증
- 계정/패스워드 기반 인증
- 사용자별 고유 ID
- 권한 분리 지원

### 3.2 인증 실패 대응
- 5회 이하 실패 시 계정 잠금
- 관리자 알림 제공

## 4. 데이터 보호

### 4.1 전송 데이터 보호
- 관리접속은 반드시 암호화 통신 사용

### 4.2 저장 데이터 보호
- 패스워드, 암호키 암호화 저장
- salt 48bit 이상, iteration 1000회 이상
